title: "MCP Server"
summary: "Connect anon.li to AI assistants like Claude using the Model Context Protocol."
---

The anon.li MCP server lets AI assistants manage your email aliases, query drops, and check account usage through the [Model Context Protocol](https://modelcontextprotocol.io). It works with Claude.ai, Claude Desktop, Claude Code, and any MCP-compatible client.

## Setup

### Prerequisites

You'll need an anon.li API key. Generate one in your [Dashboard](/dashboard/api-keys) — it will start with `ak_`.

### Claude Connectors

[Claude Connectors](https://docs.anthropic.com/en/docs/agents-and-tools/claude-code/claude-code-mcp-servers) let you add MCP tools to Claude.ai, the Anthropic API, and Claude for Enterprise — no local setup needed.

<Step step={1} title="Get your API key">

Generate an API key in your [Dashboard](/dashboard/api-keys). It will start with `ak_`.

</Step>

<Step step={2} title="Add the connector">

In the [Anthropic Console](https://console.anthropic.com), go to **Settings > Connectors** and create a new connector:

- **URL**: `https://anon.li/api/mcp`
- **Authentication**: Bearer Token
- **Token**: Your anon.li API key (`ak_...`)

</Step>

<Step step={3} title="Use it">

The anon.li tools will be available in Claude.ai and API calls for your organization. Claude can now create aliases, manage drops, and more on your behalf.

</Step>

### Claude Desktop

Add this to your Claude Desktop configuration file (`claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "anon.li": {
      "type": "streamable-http",
      "url": "https://anon.li/api/mcp",
      "headers": {
        "Authorization": "Bearer ak_your_api_key_here"
      }
    }
  }
}
```

### Claude Code

```bash
claude mcp add --transport http anon.li https://anon.li/api/mcp
```

## Available Tools

### Aliases

| Tool | Description |
|------|-------------|
| `list_aliases` | List all email aliases with forwarding stats |
| `create_alias` | Create a new random or custom email alias |
| `get_alias` | Get details for a specific alias |
| `update_alias` | Toggle active state, change label/note, or update recipients |
| `delete_alias` | Permanently delete an alias |

### Drops

| Tool | Description |
|------|-------------|
| `list_drops` | List your encrypted file drops with pagination |
| `get_drop` | Get drop metadata and file list |
| `delete_drop` | Permanently delete a drop and its files |
| `toggle_drop` | Enable or disable a drop's download link |

### Recipients

| Tool | Description |
|------|-------------|
| `list_recipients` | List email recipients with verification status |
| `create_recipient` | Add a new forwarding destination (sends verification email) |

### Domains

| Tool | Description |
|------|-------------|
| `list_domains` | List custom domains with DNS verification status |
| `get_domain` | Get domain details and required DNS records |
| `verify_domain` | Trigger DNS record verification |

### Account

| Tool | Description |
|------|-------------|
| `get_account` | Get plan tier, usage stats, and limits |

## Example Prompts

Once connected, you can ask Claude things like:

- "Create a new random alias for newsletter signups"
- "List my aliases and show which ones have received the most emails"
- "Disable the alias I created for that sketchy website"
- "How much storage am I using?"
- "Check if my custom domain DNS records are verified"
- "Show me all my drops and when they expire"

## Limitations

- **Drop creation/upload is not available** via MCP. Drops use end-to-end encryption that requires client-side key generation - use the [CLI](/docs/cli) or [web UI](https://anon.li) to create drops.
- **Drop contents are encrypted** and cannot be read through the MCP server. The server can only manage drop metadata (list, delete, toggle).
